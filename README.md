# 🏦 AI4Fintech — Каталог AI Use-cases

Интерактивный фронтенд-каталог AI-решений для банков и финтех-компаний. Проект представляет собой статическую HTML-страницу (index.html) с набором use-cases, фильтрами и поиском по категориям.

## Что это

- Одна страница (SPA) на чистом HTML/CSS/JavaScript, готовая к размещению на GitHub Pages или любом статическом хостинге.
- Каталог содержит большое количество бизнес-кейсов для банковских процессов: кредитование, AML/комплайенс, фрод-детекцию, цифровой банкинг, операции, риск и т.д.

## Файлы

- `index.html` — основная страница: интерфейс, список use-cases (встроён в JS), фильтры, поиск и модальные окна.
- `.github/` — (опционально) настройки CI/ISSUE templates и т.п.

## Запуск локально

1. Откройте `index.html` в браузере — страница полностью статическая, backend не требуется.
2. Для разработки удобно запускать простой локальный HTTP-сервер из корня репозитория:

```bash
# Python 3
python -m http.server 8000
# затем открыть http://localhost:8000
```

## Развёртывание

- GitHub Pages: включите Pages в настройках репозитория и укажите ветку `main` (или другую, если используется).
- Любой статический хостинг: Netlify, Vercel (static), Surge и т.п.

## Как дополнять каталог

- Список use-cases находится прямо в `index.html` в переменной `data` внутри `<script>`.
- Для добавления новых кейсов — добавьте объект с полями: `category`, `title`, `description` (можно добавить `type: 'main'|'additional'` если нужно разделять).

Пример элемента:

```js
{ category: "Кредитование ЮЛ", title: "Анализ кредитных документов ЮЛ", description: "Автоматическое извлечение и проверка документов..." }
```

## Стиль и доступность

- Простая адаптивная сетка; поддержка мобильных экранов через CSS media queries.
- Улучшения: добавить lazy-loading для больших списков, поиск на стороне сервера (если каталог разрастётся), i18n.

## Планы и улучшения

- Вынос данных в JSON/MD файлы для упрощения редактирования.
- Добавление метаданных (tags, maturity, implementation notes).
- Поддержка экспортов (CSV/JSON) списка выбранных use-cases.
